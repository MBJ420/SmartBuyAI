Frontend for Python Devs

A visual guide to understanding the SmartBuyAI frontend code.

1. The Big Picture: How it all fits together

Since you are building a web app, the browser only understands three languages. Here is what they do in the context of SmartBuyAI:

HTML = The Skeleton

The text boxes where users type "laptop" and "50000". The physical "Search" button on the screen.

CSS = The Makeup

Making the button blue, the text boxes rounded, and positioning everything neatly in the center of the screen.

JavaScript = The Muscle

Detecting when the button is clicked, grabbing the text from the boxes, and sending it to your Python backend.

2. The Core Function: Taking the user's order to Python

In our project plan, we have a function called searchProducts(). This is the only piece of JavaScript you really need to understand to connect your frontend to your backend.

Let's look at the exact JavaScript code we will use, side-by-side with how you would write the exact same logic in Python.

JavaScript (Frontend)
async function searchProducts() {
    // 1. Read what the user typed in the HTML
    const product = document.getElementById("productInput").value;
    const budget = document.getElementById("budgetInput").value;

    // 2. Send request to Python FastAPI
    const url = `http://localhost:8000/search?product=${product}&budget=${budget}`;
    const response = await fetch(url);
    
    // 3. Parse the JSON returned by Python
    const data = await response.json();

    // 4. Print it to the browser console
    console.log(data); 
}
Python (Mental Model Equivalent)

3. The Translation Guide (Line by Line)
A. Defining Variables

In Python, you just type name = "laptop". In JS, you must put const or let in front of it.

  • const = A variable that will never change (Constant). Like the user's search term.
  • let = A variable that might change later (like a counter in a loop).
B. Talking to the HTML (The DOM)
document.getElementById("productInput").value

This is the most famous line in JavaScript.
document represents your entire HTML page.
getElementById is a built-in function that searches the HTML for a specific tag (e.g., <input id="productInput">).
.value extracts the text the user actually typed inside that box.

C. Formatting Strings (f-strings vs backticks)

In Python, you use f"Hello {name}".
In JavaScript, you use backticks (the key above Tab) and a dollar sign: `Hello ${name}`. This allows us to inject the user's product and budget into the URL.

D. Making the API Call (fetch vs requests)
const response = await fetch(url);

In Python, you use the requests library to download a webpage or hit an API. In JavaScript, that tool is built directly into the browser and is called fetch(). It sends the HTTP request to your FastAPI backend.

Why await? Talking to a server takes time (maybe 1 or 2 seconds). If JavaScript doesn't wait, it will move to the next line of code before the data arrives, causing an error. await tells JS: "Freeze this function right here until the server responds."

E. Reading the Data
const data = await response.json();

When FastAPI sends data back, it sends it as a raw text string. .json() parses that text into a JavaScript Object (which looks and acts exactly like a Python Dictionary). Now you can do things like data[0].price.

4. What happens after we get the data?

Right now, the code just says console.log(data). This prints the data to the browser's hidden developer console (Press F12 in Chrome to see it). It doesn't show it to the user yet.
